# Changelog

All notable changes to the A2A Commerce Specification will be documented in this file.

The format is based on [Keep a Changelog](https://keepachangelog.com/en/1.1.0/),
and this specification adheres to [Semantic Versioning](https://semver.org/spec/v2.0.0.html).

## [v1.0] — 2026-04-11

### Added
- Initial public release of the A2A Commerce Specification.
- Formal definition of A2A Commerce as the fifth paradigm of electronic commerce.
- AI Commerce Maturity Ladder (ACML) — 6-level framework (L0 through L5).
- Seven core principles: Agent First, Principal Sovereignty, Protocol Neutral, Transparent Negotiation, Zero Trust Security, Fair Economics, Auditable Trail.
- Transaction lifecycle (10 phases) and negotiation protocol definition.
- Agent identity and trust framework with Agent Card schema.
- Interoperability standards covering REST, MCP, Google A2A Protocol, GraphQL, and gRPC.
- Economic model and platform revenue structure.
- Governance framework with four-level dispute resolution.
- Explicit differentiation from Google's A2A Protocol.
- Legal compliance and ethical guidelines.
